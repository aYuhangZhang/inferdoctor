# Internationalization

InferDoctor currently has first-step internationalization for the health dashboard and `inferdoctor check` console summary.

Supported language values:

- `auto`
- `en`
- `zh`
- `ja`

Example:

```bash
inferdoctor --language zh
inferdoctor --language ja
inferdoctor check --language en
```

## Current Scope

Localized today:

- default health dashboard
- `inferdoctor check` console summary
- dashboard labels for PASS / WARN / FAIL / SKIP summaries

May remain English today:

- non-dashboard command output
- generated starter templates
- JSON field names
- Markdown report schema fields
- machine-readable performance report schemas
- example app source code

This is intentional for the first i18n step. Machine-readable schemas stay language-neutral and stable so issue reports, automation, and saved reports do not break across languages.

## Auto Detection

`--language auto` follows common locale environment variables when possible:

- `LANGUAGE`
- `LC_ALL`
- `LC_MESSAGES`
- `LANG`

Neutral values such as `C`, `POSIX`, and `C.UTF-8` fall back to English.

Unsupported language values are rejected so mistakes are visible.

## Contribution Areas

Helpful contributions:

- Japanese wording for local AI performance terminology
- Chinese wording for local AI performance terminology
- clearer translations for health dashboard labels
- examples that explain TTFT, streaming, TPS, and RAG latency naturally in Chinese or Japanese
- tests that assert real rendered localized output

Please keep translations concise. CLI output should stay readable in screenshots and terminal logs.

## Terms That Need Care

Performance terms should be translated carefully:

- TTFT: time to first token
- TPS: tokens per second
- streaming
- cold start
- warmup
- endpoint
- OpenAI-compatible endpoint
- RAG retrieval latency
- rerank latency
- context budget
- smoke test, not benchmark

When a term is commonly used in English by developers, keeping the English acronym with a short explanation is acceptable.
