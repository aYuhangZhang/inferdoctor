# InferDoctor v0.4.0 - Beginner-Friendly Local AI App Setup

InferDoctor v0.4.0 is a product-quality pass on the local AI setup-assistant workflow. It focuses on making the existing v0.4 development work easier to understand, safer to try, and more useful for beginners before any official release, main merge, or PyPI publish.

## What Changed

- Polished beginner setup journeys from `inferdoctor` to generated template validation and smoke testing.
- Improved generated starter app messaging so dry-run and config-check modes are obvious before a live endpoint is used.
- Added clearer next-step guidance for `inferdoctor recommend`, `inferdoctor stack bootstrap --dry-run`, `inferdoctor template create`, and `inferdoctor template validate`.
- Added golden demo outputs under `examples/demo_outputs/` for README, launch, and documentation use.
- Updated the README first screen around the setup-assistant workflow.

## Why It Matters

Local AI is powerful, but setup can be confusing. A beginner may not know whether the problem is CUDA, a stopped runtime, a wrong `/v1` base URL, a model that does not fit in memory, or an app template that is not configured yet.

v0.4.0 tightens the practical path:

```bash
inferdoctor
inferdoctor recommend --goal customer-service
inferdoctor stack plan --goal customer-service
inferdoctor stack bootstrap --goal customer-service --dry-run
inferdoctor template create customer-service --output ./customer-service-demo
inferdoctor template validate ./customer-service-demo
inferdoctor template smoke-test ./customer-service-demo
```

## Dry-Run Template Experience

Generated starter projects include safe commands such as:

```bash
python app.py --dry-run
python app.py --check-config
python query.py --dry-run
python query.py --check-config
```

These commands do not contact a model endpoint. They help users inspect settings, prompts, sample data, and retrieval behavior before using a live local runtime.

## Template Smoke-Test

`inferdoctor template smoke-test <path>` checks that a generated project can run safe help, dry-run, and config-check commands. It does not install dependencies, start services, call endpoints, or run model inference.

## Stack Bootstrap Dry-Run

`inferdoctor stack bootstrap --goal customer-service --dry-run` shows the careful beginner path without executing installer commands or starting runtimes. It explains what is safe, what requires explicit approval, and what InferDoctor will not do automatically.

## Safety Defaults

InferDoctor remains lightweight and safe by default:

- It does not install Ollama, vLLM, SGLang, Xinference, CUDA, Docker, or other runtimes.
- It does not download models.
- It does not run model inference during diagnosis, validation, smoke testing, or bootstrap dry-run.
- It does not modify system settings.
- Template generation writes only to the output directory chosen by the user.

## Install

For current users, install the latest published package from PyPI:

```bash
pip install inferdoctor
```

For the historical v0.4.0 source tag:

```bash
python -m pip install "git+https://github.com/anguoyang/inferdoctor.git@v0.4.0"
```

For local development:

```bash
git clone https://github.com/anguoyang/inferdoctor.git
cd inferdoctor
python -m pip install -e ".[dev]"
```

## Known Limitations

- v0.4.0 was prepared before the first PyPI publication; v0.4.1 and later can be installed from PyPI.
- Recommendations and model-fit estimates are rule-based heuristics, not benchmarks.
- Template smoke tests check startup behavior only; they do not prove a live model endpoint is healthy.
- Generated templates are starter examples, not production-ready deployments.

## Upgrade Notes

- Existing v0.3 commands remain available.
- No heavy AI runtime dependency was added.
- No default behavior changes from read-only diagnosis to automatic setup.
- Users should run `inferdoctor template smoke-test` after generating a starter project.
