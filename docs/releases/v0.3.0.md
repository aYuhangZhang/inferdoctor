# InferDoctor v0.3.0 - Local AI Setup Assistant

InferDoctor v0.3.0 expands the project from a local AI stack checker into a local AI stack doctor and setup assistant.

The goal is simple: help developers diagnose their machine, understand what it can realistically run, choose a reasonable local AI stack, generate a starter project, validate that project, and follow clear next steps toward a working local AI application.

## What Changed

v0.3.0 adds a setup-assistant workflow on top of the existing health checks:

- Template catalog for common local AI application goals.
- Starter project generation for customer service, restaurant ordering, and local document Q&A demos.
- Template validation for generated projects.
- Guided setup with `inferdoctor init`.
- Hardware-aware stack recommendations with `inferdoctor recommend`.
- Beginner-friendly stack planning with `inferdoctor stack plan`.
- Heuristic model fit guidance with `inferdoctor model fit`.
- Improved generated template READMEs, config files, prompts, sample data, and troubleshooting notes.
- Beginner documentation for local AI stacks, OpenAI-compatible endpoints, templates, recommendations, and model fit.

## Why It Matters

Local AI is powerful, but beginners often get stuck before they can build anything useful. The problem is rarely one single error. It may be the GPU driver, CUDA toolkit, runtime endpoint, model server URL, Docker daemon, Dify/Open WebUI configuration, or a mismatch between hardware and model size.

InferDoctor now helps answer two questions:

1. Why does my local AI stack not work?
2. What can I build on this machine, and how do I get started?

## New Commands

```bash
inferdoctor template list
inferdoctor template show customer-service
inferdoctor template create customer-service --output ./customer-service-demo
inferdoctor template validate ./customer-service-demo

inferdoctor init --goal customer-service --preference easiest
inferdoctor recommend --goal customer-service --vram 24
inferdoctor stack plan --goal customer-service --vram 24
inferdoctor model fit --size 14b --quant q4 --vram 24
```

## Starter Templates

v0.3.0 includes starter projects for:

- `customer-service`: FAQ-style assistant for support workflows.
- `restaurant-ordering`: menu and ordering-policy assistant.
- `local-doc-qa`: lightweight Markdown document Q&A with keyword fallback.

Generated templates use local OpenAI-compatible endpoints by default and include `.env.example`, `config.yaml`, sample data, prompts, README instructions, and troubleshooting notes.

## Safety

InferDoctor remains lightweight and safe by default:

- It does not install AI runtimes by default.
- It does not download models by default.
- It does not run model inference during diagnostics or template validation.
- It does not modify system settings.
- Template generation writes only to the output directory you choose.
- Recommendation and model-fit output is heuristic guidance, not a benchmark.

## Install

If PyPI publishing has not happened yet, install from GitHub:

```bash
python -m pip install "git+https://github.com/anguoyang/inferdoctor.git@dev"
```

For local development:

```bash
git clone https://github.com/anguoyang/inferdoctor.git
cd inferdoctor
python -m pip install -e ".[dev]"
```

## Upgrade Notes

- Existing `inferdoctor check`, `inferdoctor report`, `inferdoctor explain`, `inferdoctor scenario`, `inferdoctor profile`, and `inferdoctor capacity` commands remain available.
- v0.3.0 adds new setup-assistant commands without adding heavy AI runtime dependencies.
- Generated projects are examples. Review and adapt them before using them in production.

## Known Limitations

- Recommendations are rule-based heuristics, not benchmarks.
- Model fit estimates do not account for every runtime setting, context length, KV cache policy, CPU offload mode, or model architecture.
- Template validation checks project structure and obvious configuration issues, but it does not contact a live model endpoint.
- Templates are intentionally lightweight and do not include heavy vector databases, hosted services, or production deployment automation.
